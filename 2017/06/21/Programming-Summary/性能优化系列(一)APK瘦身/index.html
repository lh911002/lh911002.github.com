性能优化系列(一)APK瘦身

发表于 2017-06-21 | 分类于 Android ， 优化 | | 阅读次数:

1. 背景

APK的大小直接影响用户使用APP的第一体验，是给用户留下的第一个印象。

2. 瘦身的原则

  • 移除没有使用的资源：移除那些从来没有使用的资源；
  • 优化很少使用的资源：对于那些很少使用的资源可以考虑其它的替代和实现方式，比如对于项目中很少使用的一个第三方控件可以考虑使用自己实现的方式替换，最终将很少使用的资源也移除掉。
  • 优化必须使用的资源：对于必须使用的资源，采用压缩等优化方式降低占用空间；

3. 具体实践

说到底，要对APK进行瘦身，我们可以从哪些方面入手？

使用Android studio 的 Build->Analyze APK功能分析APK的构成如下图：

可以看出影响apk的主要因素：

  • so文件
  • dex文件
  • res，主要是图片

3.1 so文件

  • 启用abiFilter
  • 只使用一个CPU架构，推荐armeabi-v7a，覆盖范围广，并且相对于armeabi对性能有优化

3.2 dex优化

  • 减少不必要的依赖，合并功能重叠的依赖
  • proguard混淆规则优化
  • redex，redex优化后可能会发生崩溃，需要进行全面回归。

redex优化效果

3.3 res优化

主要是图片优化

  • 删除无用资源
  • 只使用一套图，推荐xhdpi
  • tinypng
  • AndResGuard
